# 小学生学好 Claude Code 实操指南

> 适合年龄：8-12岁，需要家长或老师陪同入门

---

## 第一部分：搞清楚 Claude Code 是什么

### 它不是聊天机器人

Claude Code 是一个住在你电脑"黑色窗口"（终端）里的 AI 助手。你用文字告诉它要做什么，它就帮你**真正地写代码、改文件、运行程序**——不只是给你建议，而是直接动手。

想象一个超级聪明的同桌：你说"帮我写一个猜数字游戏"，它马上就写好了，你运行一下就能玩。

### 你需要提前装好的东西

| 需要什么 | 怎么检查已装好 |
|---|---|
| Node.js（18版本以上） | 打开终端，输入 `node -v`，看到版本号就行 |
| npm | 输入 `npm -v` |
| Claude Code | 输入 `claude --version` |

**装 Claude Code 的命令（让家长帮你在终端里输入）：**
```bash
npm install -g @anthropic-ai/claude-code
```

## 第二部分：第一次动手——10分钟上手

### 步骤1：打开终端

- **Mac**：按 `Command + 空格`，搜索"终端"，回车
- **Windows**：按 `Win键`，搜索"cmd"或"PowerShell"，回车

### 步骤2：建一个自己的项目文件夹

```bash
mkdir 我的第一个项目
cd 我的第一个项目
```

### 步骤3：启动 Claude Code

```bash
claude
```

你会看到一个提示符在等你说话。

### 步骤4：说第一句话

试试输入这句话：
```
帮我写一个Python程序，让用户输入自己的名字，然后打印"你好，[名字]！今天也要加油哦！"
```

Claude Code 会直接在你的文件夹里创建文件。然后你输入：
```bash
python hello.py
```

程序跑起来了！这就是 Claude Code 的基本工作方式。

## 第三部分：学会好好"说话"——提示词技巧

Claude Code 能做多好，80% 取决于你怎么说。

### 原则1：说清楚"是什么"，而不只是"做什么"

| 差的说法 | 好的说法 |
|---|---|
| 帮我做个游戏 | 帮我用Python做一个猜数字游戏，电脑随机出1到100之间的数，我来猜，每次告诉我猜大了还是猜小了，猜对了告诉我用了几次 |
| 帮我改一下 | 第12行的变量名 `a` 改成 `player_score`，让代码更容易看懂 |

### 原则2：分步骤来，不要一次全说

**错误示范（一次说太多）：**
```
帮我做一个有登录页面、排行榜、音效、存档功能的猜数字游戏
```

**正确做法（一步一步来）：**
```
第一步：先帮我做最基本的猜数字游戏，能运行就行
（等做好后）
第二步：加一个计分功能，记录我猜了几次
（等做好后）
第三步：把分数存到文件里，下次还能看到
```

### 原则3：看不懂就直接问

```
你刚才写的第5行，`for i in range(10)` 是什么意思？用小学生能懂的方式解释
```

Claude Code 会用你能理解的语言解释。

### 原则4：出错了这样说

程序报错时，把错误信息直接粘贴给它：
```
我运行之后出现这个错误，帮我看看是什么问题：
NameError: name 'score' is not defined
```

## 第四部分：5个必学的实际操作

### 操作1：让它解释别人的代码

把你不懂的代码粘贴进去，说：
```
帮我解释这段代码每一行是做什么的，我是小学生，用简单的话说
```

### 操作2：让它帮你找错误

```
这段代码运行结果不对，应该输出10但实际输出了0，帮我找找哪里错了：
[把代码粘贴进来]
```

### 操作3：让它帮你改进代码

```
这段代码能用但很乱，帮我整理一下，加上中文注释，让我以后能看懂
```

### 操作4：让它教你一个新概念

```
我想学"循环"是什么意思，帮我写一个有趣的例子，然后解释它
```

### 操作5：让它帮你做作业项目

```
我要做一个期末作业：统计一篇文章里每个汉字出现了几次，用Python实现。
帮我先列出解决步骤，我来一步步学着做
```

注意：让它列步骤，**不要让它直接全部做完**，这样你才能真正学到东西。

## 第五部分：容易踩的坑 + 怎么避免

### 坑1：全让 Claude Code 做，自己不动脑

**症状**：每次都直接要答案，代码能跑但自己完全不懂。

**解决**：先自己想5分钟，写下你的思路，然后让 Claude Code 帮你检查和改进，而不是直接要答案。

### 坑2：不读 Claude Code 给的代码

**症状**：复制粘贴，出了问题完全不知道从哪下手。

**解决**：每次 Claude Code 写完代码，逐行问它"这行是干嘛的"，直到全部看懂再运行。

### 坑3：描述太模糊

**症状**：说"帮我做个好看的东西"，结果做出来完全不是你想要的。

**解决**：用"我要做一个……，它需要能……，看起来像……"的格式来描述。

### 坑4：遇到错误就放弃

**症状**：程序报错了，不知道怎么办，直接关掉。

**解决**：报错信息就是线索，把它完整粘贴给 Claude Code，说"帮我看这个错误"。

### 坑5：忘了保存和备份

**解决**：每次做完一个小功能，马上做一个备份：
```bash
cp 我的游戏.py 我的游戏_备份.py
```

## 第六部分：4周学习路线图

### 第1周：认识工具

- 每天花20分钟，让 Claude Code 帮你写3个不同的小程序
- 目标：不害怕终端，知道基本流程

**每天练习题：**
1. 写一个程序，计算你的年龄还有多少天到下一个生日
2. 写一个程序，输入两个数，输出它们的加减乘除
3. 写一个程序，打印出你最喜欢的10句话

### 第2周：看懂代码

- 每次 Claude Code 写完，必须让它逐行解释
- 自己尝试改一处地方，看看会发生什么

**每天挑战：**
- 拿到一段代码，在不运行的情况下猜它会输出什么，然后运行验证

### 第3周：自己动手改

- 先自己写（哪怕写得很烂），再让 Claude Code 帮你改进
- 不要直接要答案

**项目练习：**
做一个"个人小日记程序"：能写日记、能读以前的日记。分三天做，每天做一个功能。

### 第4周：完成一个完整项目

选一个你真正感兴趣的方向，做一个完整的小项目：

| 兴趣方向 | 项目建议 |
|---|---|
| 游戏 | 文字版冒险游戏 |
| 数学 | 口算练习程序（自动出题、判分） |
| 语文 | 成语接龙检查器 |
| 音乐 | 节拍计数器 |
| 体育 | 体能训练记录本 |

## 第七部分：和家长/老师一起用的建议

### 家长可以做的

1. **第一次一起坐下来**：陪孩子装好环境，跑通第一个程序
2. **不要替他们打字**：让孩子自己输入命令，手指记忆很重要
3. **遇到错误一起看**：不要帮他们查答案，引导他们把错误告诉 Claude Code
4. **鼓励展示**：让孩子给你"演示"他们做的程序，能讲清楚才算真的懂了

### 老师可以设计的练习

- **每周一题**：给一个有错误的程序，让学生找出问题并修复
- **合作项目**：两人一组，一人描述需求，一人和 Claude Code 沟通实现
- **代码日记**：每次学完写50字：今天做了什么、学到了什么、有什么不懂的

## 快速参考卡（可以打印出来贴桌上）

```
启动 Claude Code：  claude
退出：              /exit 或按 Ctrl+C
清屏：              /clear

说话模板：
- 帮我写一个[描述]，要求：[具体要求]
- 这行代码[粘贴代码]是什么意思？
- 我出现了这个错误[粘贴错误]，帮我看看
- 帮我把这段代码解释给小学生听

遇到问题先问自己：
1. 我说清楚我要什么了吗？
2. 我看懂它写的代码了吗？
3. 错误信息粘贴进去了吗？
```
